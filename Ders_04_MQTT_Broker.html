MQTT broker qurulumu

İndi simulyatorla baza arasına Mosquitto brokerini qoyuruq. Sensor artıq birbaşa bazaya yazmır — o, mesajı brokerə publish edir. Bu, sistemi əsl sənaye memarlığına çevirən addımdır. Mosquitto-nu konfiqurasiya edəcək, topic strukturunu quracaq və publish/subscribe testi keçirəcəyik.

01

Nə dəyişir?

Dərs 3-də simulyator birbaşa PostgreSQL-ə yazırdı. Bu, sadə idi, amma çevik deyildi. İndi arada MQTT broker olacaq:

Dərs 3 — birbaşa

Sensor → PostgreSQL

Sensor bazanın parolunu bilməli, bağlantını idarə etməli idi. Yalnız bir alıcı ola bilərdi.

Dərs 4 — broker vasitəsilə

Sensor → MQTT Broker → (çoxlu alıcı)

Sensor sadəcə «bu ədədi gördüm» deyir. Broker onu dinləyən hər kəsə paylayır: baza, dashboard, alert.

Üstünlük
Broker olduqda bir mesajı eyni anda bir neçə sistem ala bilər. Məsələn temperatur oxunuşu həm bazaya yazıla, həm dashboard-da göstərilə, həm də hədd aşılıbsa alarm yarada bilər — sensor bunların heç birini bilmədən.
02

Topic strukturu

MQTT-də mesajlar «topic» adlı kanallar üzrə axır. Zarat üçün aydın, iyerarxik struktur qururuq. Baza yol .env-də MQTT_TOPIC_BASE=zavod/siyezen kimi təyin olunub.

SensorCihazTopic
TemperaturS001zavod/siyezen/temperatur/S001
RütubətS002zavod/siyezen/rutubet/S002
ÇəkiS003zavod/siyezen/ceki/S003
VibrasiyaS004zavod/siyezen/vibrasiya/S004
EnerjiS005zavod/siyezen/enerji/S005
Wildcard (əvəzləyici) — # və +
Abunə olanda bütün sensorları bir dəfəyə dinləmək üçün zavod/siyezen/# yazırıq — # «bu yolun altındakı hər şey» deməkdir. Yalnız temperaturları dinləmək üçün zavod/siyezen/temperatur/+ — + «bir səviyyə istənilən dəyər» deməkdir.
03

Mosquitto konfiqurasiyası avtomatik yaranır

Dərs 1-də Mosquitto-nu quraşdırdıq. İndi ona konfiqurasiya faylı veririk ki, düzgün portda işləsin. Zavod brokeri standart 1883 portunda işləyəcək.

01_config/mosquitto.conf yaradır
cd ~/Desktop/Zarat_Faza2_Zavod

cat > 01_config/mosquitto.conf << 'EOF'
# ============================================================
# Zarat Faza 2 — Mosquitto MQTT broker konfiqurasiyasi
# Fayl: 01_config/mosquitto.conf
# ============================================================

# dinleme portu (standart MQTT)
listener 1883 localhost

# inkishaf merhelesinde anonim qosulmaya icaze
# (istehsalatda auth elave edilecek — ders 10)
allow_anonymous true

# davamliliq: broker soyunse mesajlar diskde qalsin
persistence true
persistence_location /tmp/mosquitto_zarat/

# jurnal
log_type warning
log_type notice
log_type information
EOF

# davamliliq qovlugunu yaradaq
mkdir -p /tmp/mosquitto_zarat

echo "mosquitto.conf yaradildi:"
ls -la 01_config/mosquitto.conf
Brokeri işə sal

İndi brokeri bu konfiqurasiya ilə işə salırıq. Bu, ayrı bir Terminal pəncərəsində daim açıq qalmalıdır (broker işləyən müddətdə mesajları daşıyır).

04

Broker işləyirmi? — komanda xətti testi

Python-a keçməzdən əvvəl brokerin işlədiyini Mosquitto-nun öz alətləri ilə yoxlayaq. İki yeni Terminal pəncərəsi lazımdır.

Terminal B — dinləyici

subscribe
mosquitto_sub -h localhost \
  -p 1883 \
  -t "zavod/siyezen/#" -v

Bu, bütün zavod topic-lərini dinləyir və gözləyir.

Terminal C — göndərici

publish
mosquitto_pub -h localhost \
  -p 1883 \
  -t "zavod/siyezen/temperatur/S001" \
  -m '{"cihaz_kod":"S001","qiymet":24.5}'

Bir test mesajı göndərir.

Terminal C-də göndərəndə, Terminal B-də dərhal görünməlidir:

zavod/siyezen/temperatur/S001 {"cihaz_kod":"S001","qiymet":24.5}
Uğur əlaməti
Terminal B mesajı dərhal göstərdisə — broker işləyir, publish/subscribe zənciri qurulub. Bu, MQTT-nin «poçt şöbəsi» rolunun canlı sübutudur.
05

Ortaq MQTT köməkçisi avtomatik yaranır

İndi Python tərəfinə keçirik. Əvvəlcə həm publisher, həm subscriber tərəfindən istifadə olunacaq ortaq bir MQTT köməkçi modulu yazırıq. Bu, .env-dən ayarları oxuyub broker bağlantısını qurur. paho-mqtt 2.x VERSION2 callback API-ni istifadə edirik.

03_src/toplama/mqtt_komekci.py yaradır
cd ~/Desktop/Zarat_Faza2_Zavod

cat > 03_src/toplama/mqtt_komekci.py << 'EOF'
"""
Ortaq MQTT komekcisi — .env-den ayarlari oxuyur.
Fayl: 03_src/toplama/mqtt_komekci.py
paho-mqtt 2.x VERSION2 callback API istifade olunur.
"""
import os
from dotenv import load_dotenv

# .env faylini oxu (layihe koku)
_KONF = os.path.join(os.path.dirname(__file__), "..", "..", "01_config", ".env")
load_dotenv(_KONF)

MQTT_HOST = os.getenv("MQTT_HOST", "localhost")
MQTT_PORT = int(os.getenv("MQTT_PORT", "1883"))
TOPIC_BASE = os.getenv("MQTT_TOPIC_BASE", "zavod/siyezen")

# sensor tipi kodu -> topic sozu
TIP_TOPIC = {
    "S001": "temperatur",
    "S002": "rutubet",
    "S003": "ceki",
    "S004": "vibrasiya",
    "S005": "enerji",
}


def cihaz_topic(cihaz_kod: str) -> str:
    """Cihaz kodu ucun tam topic yolu qaytarir."""
    soz = TIP_TOPIC.get(cihaz_kod, "diger")
    return f"{TOPIC_BASE}/{soz}/{cihaz_kod}"
EOF

echo "mqtt_komekci.py yaradildi:"
ls -la 03_src/toplama/mqtt_komekci.py
06

Publisher — simulyator MQTT-yə avtomatik yaranır

İndi Dərs 3-ün simulyatorunu MQTT-yə bağlayırıq. Bu skript sensorları oxuyur və hər oxunuşu müvafiq topic-ə publish edir. Mesaj JSON formatında olacaq.

03_src/sensorlar/mqtt_publisher.py yaradır
cd ~/Desktop/Zarat_Faza2_Zavod

cat > 03_src/sensorlar/mqtt_publisher.py << 'EOF'
"""
Simulyator -> MQTT broker publisher.
Fayl: 03_src/sensorlar/mqtt_publisher.py
Her sensor oxunusunu muvafiq topic-e JSON kimi publish edir.
Ishe salmaq:  python mqtt_publisher.py
Dayandirmaq:  CTRL+C
"""
import os
import sys
import json
import time
import paho.mqtt.client as mqtt
from paho.mqtt.enums import CallbackAPIVersion

# toplama qovlugunu yola elave edek (mqtt_komekci ucun)
sys.path.insert(0, os.path.join(os.path.dirname(__file__), "..", "toplama"))

from sensorlar import butun_sensorlar
from mqtt_komekci import MQTT_HOST, MQTT_PORT, cihaz_topic


def on_connect(client, userdata, flags, reason_code, properties):
    if reason_code == 0:
        print(f"Broker-e qosuldu: {MQTT_HOST}:{MQTT_PORT}")
    else:
        print(f"Qosulma alinmadi, kod={reason_code}")


def main():
    client = mqtt.Client(
        callback_api_version=CallbackAPIVersion.VERSION2,
        client_id="zarat-publisher",
    )
    client.on_connect = on_connect
    client.connect(MQTT_HOST, MQTT_PORT, keepalive=60)
    client.loop_start()

    sensorlar = butun_sensorlar()
    print("Publisher başladi. CTRL+C ile dayandirin.\n")

    try:
        while True:
            for s in sensorlar:
                o = s.oxu()
                topic = cihaz_topic(o.cihaz_kod)
                payload = json.dumps(o.dict())
                client.publish(topic, payload, qos=0)
                print(f"-> {topic}  {payload}")
            print("-" * 60)
            time.sleep(2)
    except KeyboardInterrupt:
        print("\nPublisher dayandirilir...")
        client.loop_stop()
        client.disconnect()
        print("Bitdi.")


if __name__ == "__main__":
    main()
EOF

echo "mqtt_publisher.py yaradildi:"
ls -la 03_src/sensorlar/mqtt_publisher.py
VERSION2 detalı
Diqqət edin: mqtt.Client(callback_api_version=CallbackAPIVersion.VERSION2, ...). paho-mqtt 2.x-də bu vacibdir — köhnə Client(client_id) yazılışı xəta verər. on_connect imzası da yenidir: 5 arqument (...reason_code, properties).

zavod/siyezen/temperatur/S001 {"cihaz_kod": "S001", "olcme_vaxti": "...", "qiymet": 24.19} zavod/siyezen/rutubet/S002 {"cihaz_kod": "S002", ...} ...
Uğur əlaməti — bütöv zəncir işləyir
Publisher göndərir → broker qəbul edir → dinləyici alır. Bu, sensor məlumatının artıq MQTT üzərindən axdığının sübutudur. Sensor kimin dinlədiyini bilmir — bu, «loosely coupled» memarlığın gücüdür.

topics.md — topic sxemini sənədləşdirək avtomatik yaranır

Dərs 1-də 01_config/topics.md boş faylını yaratmışdıq. İndi onu dolduraq ki, komanda topic strukturunu bir yerdə görsün.

01_config/topics.md yaradır
cd ~/Desktop/Zarat_Faza2_Zavod

cat > 01_config/topics.md << 'EOF'
# Zarat Faza 2 — MQTT Topic Sxemi

Baza yol: zavod/siyezen

## Sensor topic-leri

| Cihaz | Sensor      | Topic                             |
|-------|-------------|-----------------------------------|
| S001  | Temperatur  | zavod/siyezen/temperatur/S001     |
| S002  | Rutubet     | zavod/siyezen/rutubet/S002        |
| S003  | Ceki        | zavod/siyezen/ceki/S003           |
| S004  | Vibrasiya   | zavod/siyezen/vibrasiya/S004      |
| S005  | Enerji      | zavod/siyezen/enerji/S005         |

## Wildcard-lar (abune ucun)

- zavod/siyezen/#              -> butun sensorlar
- zavod/siyezen/temperatur/+   -> yalniz temperaturlar
- zavod/siyezen/+/S001         -> yalniz S001 cihazi

## Mesaj formati (JSON)

{
  "cihaz_kod": "S001",
  "olcme_vaxti": "2026-07-09T18:30:12+00:00",
  "qiymet": 24.19
}

## QoS

- Sensor olcmeleri: QoS 0 (itse problem deyil, davamli axir)
- Xeberdarliqlar (ders 6): QoS 1 (mutleq catmalidir)
EOF
