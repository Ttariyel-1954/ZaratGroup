01 — Motivasiya

Körpünün üç zəifliyi

Dərs 04-B-də yazdığımız mqtt_kopru.py işləyir. Amma onu real zavodda 7/24 işlətmək olmaz. Üç ciddi problemi var — və FastAPI hər üçünü həll edir.

Zəiflik 2 — Hər mesajda yeni baza sorğusu, bir bağlantı

Körpü bir baza bağlantısı ilə işləyir. O bağlantı ölsə (baza restart olsa), körpü onu bərpa etməyə çalışır — amma bu müddətdə gələn mesajlar itir. Həmçinin tək bağlantı yüksək tempdə "dar boğaz"a çevrilir.

Zəiflik 3 — "Qara qutu"dur

Körpü işləyirmi? Neçə mesaj qəbul edib? Baza əlçatandırmı? Bilmək üçün terminala baxmaqdan başqa yol yoxdur. Real serverdə terminal olmur — servis fonda işləyir. Ona sual vermək imkanı lazımdır.

FastAPI nə gətirir ① Validasiya deklarativ olur — nəyin düzgün olduğunu bir dəfə təsvir edirsiniz, yoxlamanı Pydantic edir və dəqiq xəta mesajı qaytarır.
② Bağlantı hovuzu — 10 hazır bağlantı, biri ölsə digəri işləyir.
③ Servis sual verilə bilən olur — /health, /metrikalar endpoint-ləri ilə hər an vəziyyəti öyrənmək olar.
④ Bonus: avtomatik interaktiv sənədləşmə (Swagger) — gateway inteqratoruna "budur API-miz" deyib link verirsiniz.

02 — Alət

FastAPI: nədir və niyə məhz o?

FastAPI — Python üçün müasir veb-servis çərçivəsidir. İki şeyin birləşməsidir: Starlette (sürətli async veb qatı) + Pydantic (tip əsaslı validasiya). Bu birləşmə onu IoT/data servisləri üçün ideal edir.

04 — Validasiya

Pydantic: sistemin qapıçısı

Bu, Dərs 05-in ürəyidir. Pydantic modeli — "düzgün ölçmə nədir?" sualının rəsmi, maşın-oxunaqlı cavabıdır.

03_src/api/modeller.py
"""Pydantic modelləri — sistemə girən/çıxan datanın rəsmi tərifi."""
from datetime import datetime, timezone, timedelta
from typing import Optional, Literal

from pydantic import BaseModel, Field, field_validator, ConfigDict


# ============ GİRİŞ: ölçmə ============
class OlcmeIn(BaseModel):
    """Sensordan gələn bir ölçmə. Hər sahə ciddi yoxlanılır."""

    model_config = ConfigDict(
        extra="forbid",          # naməlum sahə varsa → RƏDD ET
        str_strip_whitespace=True, # boşluqları avtomatik təmizlə
    )

    cihaz_kod: str = Field(
        ...,                        # ... = MƏCBURİ
        min_length=2, max_length=20,
        pattern=r"^[A-Z0-9_-]+$",  # yalnız böyük hərf, rəqəm, _ və -
        description="Cihaz kodu, məs. S001",
        examples=["S001"],
    )

    qiymet: float = Field(
        ...,
        ge=-10_000, le=100_000,     # fiziki mümkün diapazon
        description="Ölçülən dəyər",
        examples=[24.317],
    )

    olcme_vaxti: datetime = Field(
        ...,
        description="ISO 8601, zaman zonası MƏCBURİ",
        examples=["2026-07-11T09:15:00+00:00"],
    )

    vahid: Optional[str] = Field(
        None, max_length=15,
        description="Opsional — yalnız məlumat üçün",
    )

    # ---- Xüsusi yoxlama 1: zaman zonası MƏCBURİDİR ----
    @field_validator("olcme_vaxti")
    @classmethod
    def zona_mecburidir(cls, v: datetime) -> datetime:
        if v.tzinfo is None:
            raise ValueError(
                "olcme_vaxti zaman zonası ilə olmalıdır "
                "(məs. 2026-07-11T09:15:00+04:00 və ya ...Z)"
            )
        return v

    # ---- Xüsusi yoxlama 2: gələcəkdən gələ bilməz ----
    @field_validator("olcme_vaxti")
    @classmethod
    def gelecek_deyil(cls, v: datetime) -> datetime:
        indi = datetime.now(timezone.utc)
        if v > indi + timedelta(minutes=5):   # 5 dəq. saat fərqinə dözüm
            raise ValueError(
                f"olcme_vaxti gələcəkdədir: {v.isoformat()}. "
                "Cihazın saatı düzgündürmü?"
            )
        return v

    # ---- Xüsusi yoxlama 3: çox köhnə data şübhəlidir ----
    @field_validator("olcme_vaxti")
    @classmethod
    def cox_kohne_deyil(cls, v: datetime) -> datetime:
        indi = datetime.now(timezone.utc)
        if v < indi - timedelta(days=7):
            raise ValueError(
                f"olcme_vaxti 7 gündən köhnədir: {v.isoformat()}"
            )
        return v


# ============ ÇIXIŞ: cavab modelləri ============

Hər qərarın izahı

extra="forbid"
Nə: modeldə olmayan sahə gəlsə, mesajı rədd edir.
Niyə bu qədər sərt: gateway inteqratoru səhvən "value" yazsa ("qiymet" əvəzinə), susqun rejimdə mesaj boş dəyərlə keçər və heç kim səhvi görməz. forbid ilə isə dərhal dəqiq xəta qayıdır: "naməlum sahə: value". Səhvi ilk gündə tapmaq, aylar sonra "niyə datalar səhvdir?" deyə axtarmaqdan yaxşıdır.
pattern=r"^[A-Z0-9_-]+$"
Nə: cihaz kodu yalnız böyük hərf, rəqəm, _, - ola bilər.
Niyə: ① Bizim standart S001 formatını qoruyur. ② SQL injection cəhdlərini ('; DROP TABLE) qapıda kəsir — baxmayaraq ki, parametrləşdirilmiş sorğu onsuz da qoruyur, iki qat müdafiə yaxşı vərdişdir.
ge=-10_000, le=100_000
Nə: qiymətin mütləq fiziki hüdudu.
Niyə bu qədər geniş: bu, sensor həddi deyil — o, bazadadır (min_hedd/max_hedd) və dinamik dəyişə bilər. Bu isə sağlamlıq süzgəcidir: kabel qopanda sensor bəzən 1e38 kimi zibil ədəd verir. Belə dəyər bazaya düşsə, bütün qrafiklər korlanır (bir nöqtə bütün oxu sıxışdırır).
zona_mecburidir
Nə: zonasız vaxtı rədd edir.
Niyə: Dərs 04-B-də danışdıq — TIMESTAMPTZ sütununa zonasız vaxt yazsanız, PostgreSQL onu server zonası ilə şərh edir. Zavod və mərkəz fərqli zonada olsa, ölçmələr səhv saata düşür və səbəbi aylar sonra tapılır. Qapıda kəsmək yeganə düzgün yoldur.
gelecek_deyil (+5 dəq. dözüm)
Nə: gələcəkdən gələn ölçməni rədd edir.
Niyə 5 dəqiqə dözüm: gateway-in saatı NTP ilə tam sinxron olmaya bilər — bir neçə saniyə/dəqiqə fərq normaldır. Amma saatlar fərq varsa, cihazın saatı səhvdir və bu, real problemdir: ölçmələr yanlış vaxta düşür, trend təhlili pozulur.
cox_kohne_deyil (7 gün)
Nə: həddindən artıq köhnə ölçməni rədd edir.
Niyə: gateway uzun müddət offline qalıb sonra "yaddaşındakı" köhnə mesajları töksə, onlar tarixçəni korlaya bilər. 7 gün — məqbul bufer.
⚠ Diqqət: bu limiti şüurlu seçin. Zavod bir ay şəbəkəsiz qalacaqsa, bu yoxlama datanızı yeyər.
examples=[...]
Nə: Swagger sənədində nümunə dəyər göstərir.
Niyə vacib: gateway inteqratoru brauzerdə API-ni açanda hazır nümunə görür və "Try it out" ilə dərhal sınayır. Sənədləşməni ayrıca yazmaq lazım gəlmir.
Pydantic-in əsl gücü Diqqət edin — biz bir sətir də if yazmadıq (üç xüsusi yoxlama istisna). Uzunluq, tip, diapazon, format — hamısı təsvirdir, kod deyil.

Nəticə: gələn pozuq mesaj üçün FastAPI avtomatik 422 statusu və dəqiq, sahə-sahə xəta cavabı qaytarır:
{"loc": ["body", "qiymet"], "msg": "Input should be less than or equal to 100000"}

Körpüdəki try/except isə sadəcə "pozuq mesaj" deyib atırdı — nə pozuqdur, bilinmirdi.

03_src/api/baza.py
"""Baza qatı — bağlantı hovuzu və SQL əməliyyatları."""
from psycopg_pool import AsyncConnectionPool
from .konfiq import EDGE_DSN

# Hovuz: 2 minimum, 10 maksimum bağlantı
# open=False → tətbiq başlayanda açacağıq (lifespan-da)
hovuz = AsyncConnectionPool(
    conninfo=EDGE_DSN,
    min_size=2,
    max_size=10,
    open=False,
    timeout=10,        # bağlantı gözləmə limiti (san.)
    max_lifetime=3600, # bağlantı 1 saatdan sonra yenilənir
)

# Hədd keşi — {cihaz_kod: (min, max, vahid)}
HEDLER: dict[str, tuple[float, float, str]] = {}


async def hedleri_yukle() -> int:
    """Cihaz hədlərini yaddaşa yükləyir. Başlanğıcda bir dəfə çağırılır."""
    sql = """
        SELECT c.kod, t.min_hedd, t.max_hedd, t.vahid
        FROM cihaz c
        JOIN sensor_tipi t ON c.sensor_tipi_kod = t.kod
        WHERE c.status = 'aktiv'
    """
    async with hovuz.connection() as conn:
        async with conn.cursor() as cur:
            await cur.execute(sql)
            HEDLER.clear()
            async for kod, mn, mx, vahid in cur:
                HEDLER[kod] = (float(mn), float(mx), vahid)
    return len(HEDLER)


async def olcme_yaz(kod: str, vaxt, qiymet: float) -> dict:
    """
    Ölçməni yazır; hədd aşılıbsa xəbərdarlıq da yaradır.
    Hər ikisi BİR tranzaksiyada — ya hamısı, ya heç biri.
    """
    mn, mx, vahid = HEDLER[kod]
    keyfiyyet = 1 if mn <= qiymet <= mx else 0
    xeber_mesaji = None

    async with hovuz.connection() as conn:
        async with conn.cursor() as cur:
            await cur.execute(
                """INSERT INTO olcme (cihaz_kod, olcme_vaxti, qiymet, keyfiyyet)
                   VALUES (%s, %s, %s, %s)""",
                (kod, vaxt, qiymet, keyfiyyet),
            )

            if qiymet > mx:
                xeber_mesaji = f"{kod}: {qiymet} {vahid} > maks {mx}"
                await cur.execute(
                    """INSERT INTO xeberdarliq
                       (cihaz_kod, olcme_vaxti, qiymet, novu, mesaj)
                       VALUES (%s, %s, %s, 'yuxari_hedd', %s)""",
                    (kod, vaxt, qiymet, xeber_mesaji),
                )
            elif qiymet < mn:
                xeber_mesaji = f"{kod}: {qiymet} {vahid} < min {mn}"
                await cur.execute(
                    """INSERT INTO xeberdarliq
                       (cihaz_kod, olcme_vaxti, qiymet, novu, mesaj)
                       VALUES (%s, %s, %s, 'asagi_hedd', %s)""",
                    (kod, vaxt, qiymet, xeber_mesaji),
                )
        # "async with conn" bloku bitəndə → avtomatik COMMIT
        # xəta olsa → avtomatik ROLLBACK

    return {"keyfiyyet": keyfiyyet, "xeberdarliq": xeber_mesaji}


async def son_olcmeler(limit: int = 20, cihaz: str | None = None) -> list:
    sql = """
        SELECT cihaz_kod, olcme_vaxti, qiymet, keyfiyyet, sync_status
        FROM olcme
        WHERE (%s IS NULL OR cihaz_kod = %s)
        ORDER BY olcme_vaxti DESC
        LIMIT %s
    """
    async with hovuz.connection() as conn:
        async with conn.cursor() as cur:
            await cur.execute(sql, (cihaz, cihaz, limit))
            setirler = await cur.fetchall()

    return [
        {"cihaz_kod": r[0], "olcme_vaxti": r[1], "qiymet": float(r[2]),
         "keyfiyyet": r[3], "sync_status": r[4]}
        for r in setirler
    ]


async def cihazlar() -> list:
    sql = """
        SELECT c.kod, c.ad, t.ad, t.vahid, t.min_hedd, t.max_hedd, c.status
        FROM cihaz c JOIN sensor_tipi t ON c.sensor_tipi_kod = t.kod
        ORDER BY c.kod
    """
    async with hovuz.connection() as conn:
        async with conn.cursor() as cur:
            await cur.execute(sql)
            setirler = await cur.fetchall()
    return [
        {"kod": r[0], "ad": r[1], "tip": r[2], "vahid": r[3],
         "min_hedd": float(r[4]), "max_hedd": float(r[5]), "status": r[6]}
        for r in setirler
    ]
AsyncConnectionPool(min_size=2, max_size=10)
Nə: əvvəlcədən açılmış bağlantılar dəsti.
Niyə hovuz: yeni PostgreSQL bağlantısı açmaq bahalıdır (~10-50 ms: TCP, autentifikasiya, sessiya qurulması). Saniyədə 50 mesaj gəlsə, bu, saniyənin yarısını yeyər. Hovuz isə hazır bağlantını anında verir.
Niyə min 2: heç bir sorğu olmasa da iki bağlantı hazır dayanır — ilk sorğu gecikmir.
Niyə maks 10: PostgreSQL-in default limiti 100-dür; digər servislərə (Shiny, psql) yer qoyuruq.
max_lifetime=3600
Nə: bağlantı 1 saatdan sonra bağlanıб yenisi açılır.
Niyə: uzun yaşayan bağlantılar "qocalır" — şəbəkə avadanlığı (firewall, NAT) onları səssizcə kəsə bilər və proqram bunu yalnız sorğu atanda öyrənir. Dövri yeniləmə bu problemi kökündən həll edir.
async with hovuz.connection()
Nə: hovuzdan bağlantı götürür, blok bitəndə avtomatik qaytarır (bağlamır!).
Bonus: blok uğurla bitsə → avtomatik COMMIT; istisna atılsa → avtomatik ROLLBACK. Körpüdə bunu əl ilə edirdik (conn.commit(), rollback()) — burada unutmaq mümkün deyil.
HEDLER (keş)
Körpüdəki eyni ideya: hədləri hər mesajda bazadan soruşmuruq. Fərq: burada /cihazlar/yenile endpoint-i ilə keşi servisi restart etmədən yeniləyə bilərik (aşağıda).

06 — Lifespan

Servisin həyat dövrü

Servis başlayanda hovuz açılmalı və hədlər yüklənməlidir; sönəndə hovuz səliqəli bağlanmalıdır. FastAPI bunun üçün lifespan mexanizmi verir.

main.py — lifespan hissəsi
from contextlib import asynccontextmanager
from fastapi import FastAPI

from .baza import hovuz, hedleri_yukle
from .konfiq import VERSIYA


@asynccontextmanager
async def lifespan(app: FastAPI):
    # ===== BAŞLANĞIC =====
    print("[API] Baza hovuzu açılır...")
    await hovuz.open()
    await hovuz.wait()          # bağlantılar hazır olana qədər gözlə

    say = await hedleri_yukle()
    print(f"[API] {say} cihazın həddi yükləndi.")
    print(f"[API] Hazırdır — versiya {VERSIYA}")

    yield   # ← BURADA SERVİS İŞLƏYİR

    # ===== SÖNMƏ =====
    print("[API] Hovuz bağlanır...")
    await hovuz.close()
    print("[API] Bitdi.")
Niyə yield? yield-dən əvvəlki kod servis başlayanda bir dəfə işləyir. yield-dən sonrakı kod servis sönəndə işləyir. Aradakı bütün müddət — servisin iş dövrüdür.

Niyə vacibdir: hovuzu modul səviyyəsində açsaydıq, bu, import zamanı baş verərdi — test yazanda, sənəd generasiya edəndə də baza bağlantısı tələb olunardı. Lifespan bunu servisin faktiki işə düşməsinə bağlayır.

Hissə B-də: MQTT client-i də məhz burada başladacağıq — servis qalxanda qoşulacaq, sönəndə səliqəli ayrılacaq.

07 — Endpoint

main.py — tam kod

03_src/api/main.py
"""Zarat Faza 2 — Zavod telemetriya API-si."""
from contextlib import asynccontextmanager
from datetime import datetime, timezone

from fastapi import FastAPI, HTTPException, Query, status
import psycopg

from .konfiq import VERSIYA
from .baza import (hovuz, HEDLER, hedleri_yukle,
                    olcme_yaz, son_olcmeler, cihazlar)
from .modeller import (OlcmeIn, OlcmeOut, CihazOut,
                        QebulCavabi, SaglamliqCavabi)

# Sadə metrika sayğacları
METRIKA = {"qebul": 0, "anomal": 0, "xeta": 0,
           "basladi": datetime.now(timezone.utc)}


@asynccontextmanager
async def lifespan(app: FastAPI):
    await hovuz.open()
    await hovuz.wait()
    say = await hedleri_yukle()
    print(f"[API] Hazır — {say} cihaz, versiya {VERSIYA}")
    yield
    await hovuz.close()
    print("[API] Söndü.")


app = FastAPI(
    title="Zarat Zavod Telemetriya API",
    description="Siyəzən yem zavodu — sensor məlumatlarının qəbulu və sorğusu",
    version=VERSIYA,
    lifespan=lifespan,
)


# ============ 1. SAĞLAMLIQ ============
@app.get("/health", response_model=SaglamliqCavabi, tags=["Sistem"])
async def saglamliq():
    """Servis və baza işləyirmi? Monitorinq üçün."""
    try:
        async with hovuz.connection() as conn:
            async with conn.cursor() as cur:
                await cur.execute("SELECT 1")
        baza_status = "ok"
    except Exception as e:
        raise HTTPException(
            status_code=status.HTTP_503_SERVICE_UNAVAILABLE,
            detail=f"Baza əlçatmazdır: {e}",
        )

    return SaglamliqCavabi(
        status="ok", baza=baza_status,
        cihaz_sayi=len(HEDLER), versiya=VERSIYA,
    )


# ============ 2. METRİKALAR ============
@app.get("/metrikalar", tags=["Sistem"])
async def metrikalar():
    """Servisin iş statistikası."""
    isleme_muddeti = (datetime.now(timezone.utc) - METRIKA["basladi"])
    return {
        "qebul_olunan":   METRIKA["qebul"],
        "anomal":         METRIKA["anomal"],
        "xeta":           METRIKA["xeta"],
        "isleme_saniye":  int(isleme_muddeti.total_seconds()),
        "hovuz_olcusu":   hovuz.get_stats(),
    }


# ============ 3. ÖLÇMƏ QƏBULU — ƏSAS ENDPOINT ============
@app.post("/olcme", response_model=QebulCavabi,
          status_code=status.HTTP_201_CREATED, tags=["Ölçmə"])
async def olcme_qebul(olcme: OlcmeIn):
    """
    Bir ölçməni qəbul edir, validasiyadan keçirir və bazaya yazır.
    Hədd aşılıbsa avtomatik xəbərdarlıq yaradır.

    Pydantic artıq formatı yoxlayıb — bura yalnız DÜZGÜN data çatır.
    """
    # Cihaz bazada tanınırmı? (Pydantic bunu bilə bilməz)
    if olcme.cihaz_kod not in HEDLER:
        METRIKA["xeta"] += 1
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=f"Naməlum cihaz: {olcme.cihaz_kod}. "
                   f"Tanınanlar: {sorted(HEDLER.keys())}",
        )

    try:
        netice = await olcme_yaz(
            olcme.cihaz_kod, olcme.olcme_vaxti, olcme.qiymet
        )
    except psycopg.Error as e:
        METRIKA["xeta"] += 1
        raise HTTPException(
            status_code=status.HTTP_503_SERVICE_UNAVAILABLE,
            detail=f"Baza xətası: {e}",
        )

    METRIKA["qebul"] += 1
    if netice["keyfiyyet"] == 0:
        METRIKA["anomal"] += 1

    return QebulCavabi(
        status="anomal" if netice["keyfiyyet"] == 0 else "qebul",
        cihaz_kod=olcme.cihaz_kod,
        qiymet=olcme.qiymet,
        keyfiyyet=netice["keyfiyyet"],
        xeberdarliq=netice["xeberdarliq"],
    )


# ============ 4. SON ÖLÇMƏLƏR ============
@app.get("/olcme/son", response_model=list[OlcmeOut], tags=["Ölçmə"])
async def olcme_son(
    limit: int = Query(20, ge=1, le=500, description="Neçə sətir"),
    cihaz: str | None = Query(None, description="Cihaz kodu ilə süz"),
):
    """Son ölçmələri qaytarır. Dashboard bunu çağıracaq."""
    return await son_olcmeler(limit=limit, cihaz=cihaz)


# ============ 5. CİHAZ SİYAHISI ============
@app.get("/cihazlar", response_model=list[CihazOut], tags=["Cihaz"])
async def cihaz_siyahisi():
    """Bütün cihazlar və onların hədləri."""
    return await cihazlar()


# ============ 6. KEŞİ YENİLƏ ============
@app.post("/cihazlar/yenile", tags=["Cihaz"])
async def hedleri_yenile():
    """
    Yeni cihaz əlavə etdikdən sonra keşi yeniləyir.
    Servisi RESTART ETMƏDƏN — bu, körpüdə mümkün deyildi.
    """
    say = await hedleri_yukle()
    return {"status": "yeniləndi", "cihaz_sayi": say,
            "cihazlar": sorted(HEDLER.keys())}

  • "Try it out" düyməsi — brauzerdən birbaşa sorğu göndərmək.

12 — Yekun

Yoxlama siyahısı və Hissə B

  • uvicorn 03_src.api.main:app --reload xətasız işə düşür.
  • /health → "cihaz_sayi": 5 qaytarır.
  • /docs brauzerdə açılır və endpoint-lər görünür.
  • Düzgün ölçmə 201 ilə qəbul olunur və bazaya düşür.
  • Anomal dəyər keyfiyyet=0 alır və xeberdarliq yaradır.
  • Beş pozuq mesajın hamısı rədd edilir (422 və ya 404).
  • Zonasız vaxt rədd olunur — körpüdə qəbul olunurdu!
  • /cihazlar/yenile keşi restart olmadan yeniləyir.
Hissə B-də nə edəcəyik Hazırda FastAPI yalnız HTTP qəbul edir — MQTT axını hələ köhnə körpüdən keçir. İki ayrı proses işləyir, bu isə lazımsızdır.

Hissə B-də:
① MQTT client-i FastAPI-nin lifespan-ına köçürürük — bir proses həm dinləyir, həm cavab verir;
② MQTT mesajını eyni Pydantic modelindən keçiririk — validasiya hər iki qapıda eyni olur;
③ Batch INSERT — mesajları dəstə ilə yazıб performansı 10-20 dəfə artırırıq;
④ Idempotentlik — QoS 1 dublikatlarını ON CONFLICT ilə həll edirik;
⑤ Servisi launchd ilə 7/24 işlək hala gətiririk;
⑥ Köhnə mqtt_kopru.py-ı arxivə göndəririk.
