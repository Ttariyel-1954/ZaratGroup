FastAPI toplama xidməti

Zəncirin son halqası. Bu xidmət broker-dən bütün sensor mesajlarını dinləyir, hər birini validasiya edir (mənfi çəki, ağlabatmayan temperatur rədd olunur) və yalnız təmiz məlumatı zavod_edge_db-yə yazır. Bununla tam avtomatik zəncir tamamlanır: sensor → MQTT → FastAPI → PostgreSQL.

01

Xidmət nə edir?

FastAPI xidmətinin iki işi var. Birincisi — arxa planda daim MQTT broker-i dinləmək. İkincisi — HTTP interfeysi təqdim etmək (sağlamlıq yoxlaması, statistika). Bu dərsdə hər ikisini qururuq.

AddımNə baş verir
1. DinləXidmət başlayanda MQTT broker-ə qoşulur, zavod/siyezen/#-ə abunə olur
2. AlSensor mesajı gələndə JSON parse olunur
3. YoxlaPydantic model məlumatı validasiya edir (tip, aralıq)
4. YazTəmiz məlumat olcme cədvəlinə INSERT olunur (sync_status=0)
5. Rədd etXarab məlumat bazaya düşmür, log-a yazılır
Niyə FastAPI, sadə skript yox?
Sadə subscriber skripti də işləyərdi. Amma FastAPI əlavə üstünlüklər verir: HTTP endpoint-lər (dashboard və mobil üçün hazır interfeys), avtomatik sənədləşmə (Swagger), və gələcəkdə autentifikasiya (Dərs 10). İndi qurduğumuz təməl bütün Faza 2 boyu genişlənəcək.
02

Validasiya — nə qəbul, nə rədd?

Xarab sensor mənasız dəyər göndərə bilər: mənfi çəki, 999°C temperatur, boş cihaz kodu. Bunlar bazanı «zibilləyər» və sonrakı analitikanı pozar. Validasiya qatı bu keşikçidir.

QaydaNümunə — RƏDDNümunə — QƏBUL
Cihaz kodu boş olmamalıcihaz_kod = ""cihaz_kod = "S001"
Qiymət ədəd olmalıqiymet = "abc"qiymet = 24.3
Fiziki aralıq (geniş sərhəd)qiymet = -9999qiymet = 24.3
Cihaz bazada mövcud olmalıcihaz_kod = "S099"cihaz_kod = "S001"
İki səviyyə
Format validasiyası Pydantic ilə (tip düzgündürmü, ədəddirmi). Məntiq validasiyası kodda (cihaz bazada varmı, dəyər fiziki aralıqdadırmı). Bu dərsdə hər ikisini qururuq. Dəqiq hədd-aşma xəbərdarlığı isə Dərs 6-nın mövzusudur — burada yalnız «tamamilə mənasız» dəyərləri rədd edirik.

"""
zavod_edge_db qosulma komekcisi.
Fayl: 03_src/toplama/db_komekci.py
"""
import os
from dotenv import load_dotenv

_KONF = os.path.join(os.path.dirname(__file__), "..", "..", "01_config", ".env")
load_dotenv(_KONF)


def edge_dsn() -> str:
    """zavod_edge_db ucun qosulma setri (DSN) qaytarir."""
    return (
        f"host={os.getenv('EDGE_DB_HOST', 'localhost')} "
        f"port={os.getenv('EDGE_DB_PORT', '5434')} "
        f"dbname={os.getenv('EDGE_DB_NAME', 'zavod_edge_db')} "
        f"user={os.getenv('EDGE_DB_USER', 'royatalibova')}"
    )

"""
Gelen sensor mesajinin validasiyasi (Pydantic v2).
Fayl: 03_src/toplama/validasiya.py
"""
from pydantic import BaseModel, field_validator


class OlcmeMesaj(BaseModel):
    """MQTT-den gelen bir olcme mesaji."""
    cihaz_kod: str
    olcme_vaxti: str
    qiymet: float

    @field_validator("cihaz_kod")
    @classmethod
    def cihaz_kod_bos_olmasin(cls, v: str) -> str:
        if not v or not v.strip():
            raise ValueError("cihaz_kod bos ola bilmez")
        return v.strip()

    @field_validator("qiymet")
    @classmethod
    def qiymet_fiziki_aralig(cls, v: float) -> float:
        # cox geniş fiziki serhed — yalniz tamamile menasiz deyerleri redd et
        # deqiq hedd-asma xeberdarligi Ders 6-dadir
        if v < -1000 or v > 100000:
            raise ValueError(f"qiymet fiziki araligdan kenardir: {v}")
        return v
EOF

Fayl 3 — əsas FastAPI xidməti avtomatik yaranır

İndi əsas fayl. Bu, MQTT subscriber-i (arxa planda thread-də) və FastAPI HTTP interfeysini birləşdirir. Xidmət başlayanda broker-ə qoşulur, mesaj gələndə validasiya edib bazaya yazır.

03_src/toplama/xidmet.py yaradır
cd ~/Desktop/Zarat_Faza2_Zavod

cat > 03_src/toplama/xidmet.py << 'EOF'
"""
FastAPI toplama xidmeti — MQTT dinleyir, validasiya edir, bazaya yazir.
Fayl: 03_src/toplama/xidmet.py
Ishe salmaq:
  uvicorn xidmet:app --host 0.0.0.0 --port 8020
"""
import os
import json
import logging
from contextlib import asynccontextmanager

import psycopg
import paho.mqtt.client as mqtt
from paho.mqtt.enums import CallbackAPIVersion
from fastapi import FastAPI
from pydantic import ValidationError

from db_komekci import edge_dsn
from validasiya import OlcmeMesaj
from mqtt_komekci import MQTT_HOST, MQTT_PORT, TOPIC_BASE

# --- jurnal ---
logging.basicConfig(level=logging.INFO,
                    format="%(asctime)s  %(levelname)s  %(message)s")
log = logging.getLogger("toplama")

# --- statistika (sade sayğaclar) ---
stat = {"qebul": 0, "yazildi": 0, "redd": 0}

# --- baza qosulmasi (bir defe acilir, tekrar istifade) ---
_db = None


def db_qosul():
    global _db
    if _db is None or _db.closed:
        _db = psycopg.connect(edge_dsn(), autocommit=True)
    return _db


def olcme_yaz(m: OlcmeMesaj):
    """Tesdiqlenmiş olcmeni bazaya yazir."""
    conn = db_qosul()
    with conn.cursor() as cur:
        cur.execute(
            """INSERT INTO olcme (cihaz_kod, olcme_vaxti, qiymet)
               VALUES (%s, %s, %s)""",
            (m.cihaz_kod, m.olcme_vaxti, m.qiymet),
        )


# ================= MQTT callback-lari (VERSION2) =================
def on_connect(client, userdata, flags, reason_code, properties):
    if reason_code == 0:
        topic = f"{TOPIC_BASE}/#"
        client.subscribe(topic, qos=0)
        log.info(f"Broker-e qosuldu, abune: {topic}")
    else:
        log.error(f"Broker qosulmasi alinmadi: {reason_code}")


def on_message(client, userdata, msg):
    stat["qebul"] += 1
    try:
        data = json.loads(msg.payload.decode())
        m = OlcmeMesaj(**data)          # <-- format validasiyasi
        olcme_yaz(m)                    # <-- bazaya yazma
        stat["yazildi"] += 1
    except (ValidationError, ValueError) as e:
        stat["redd"] += 1
        log.warning(f"REDD ({msg.topic}): {e}")
    except Exception as e:
        stat["redd"] += 1
        log.error(f"XETA ({msg.topic}): {e}")


# ================= MQTT clienti =================
mqtt_client = mqtt.Client(
    callback_api_version=CallbackAPIVersion.VERSION2,
    client_id="zarat-toplama",
)
mqtt_client.on_connect = on_connect
mqtt_client.on_message = on_message


# ================= FastAPI lifespan =================
@asynccontextmanager
async def lifespan(app: FastAPI):
    # baslanğic: broker-e qosul, dinlemeye başla
    mqtt_client.connect(MQTT_HOST, MQTT_PORT, keepalive=60)
    mqtt_client.loop_start()
    log.info("Toplama xidmeti başladi.")
    yield
    # bitiş: temiz bağla
    mqtt_client.loop_stop()
    mqtt_client.disconnect()
    log.info("Toplama xidmeti dayandi.")


app = FastAPI(title="Zarat Faza 2 — Toplama Xidmeti", lifespan=lifespan)


@app.get("/")
def kok():
    return {"xidmet": "Zarat toplama", "veziyyet": "işleyir"}


@app.get("/saglamliq")
def saglamliq():
    return {"status": "ok", "statistika": stat}


@app.get("/statistika")
def statistika():
    return stat
EOF

Validasiyanı sınayaq — xarab mesaj

Validasiya həqiqətən işləyirmi? Əl ilə xarab bir mesaj göndərək və rədd olunduğunu görək. Xidmət (C) və broker (B) işləyən halda, yeni Terminal-da:

  -t "zavod/siyezen/temperatur/S001" \
  -m '{"cihaz_kod":"S001","olcme_vaxti":"2026-07-09T18:00:00+00:00","qiymet":-99999}'

# boş cihaz kodu — validasiya redd etmeli
mosquitto_pub -h localhost -p 1883 \
  -t "zavod/siyezen/temperatur/S001" \
  -m '{"cihaz_kod":"","olcme_vaxti":"2026-07-09T18:00:00+00:00","qiymet":24}'
